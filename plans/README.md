# plans/

Эта папка хранит планы задач и handoff-файлы между сессиями.

## Файлы

| Шаблон имени | Что это |
|---|---|
| `YYYY-MM-DD-feature-name.md` | План реализации фичи (создаёт architect) |
| `handoff.md` | Прогресс текущей сессии (создаёт `/handoff`) |

## Как пользоваться

### Начало новой задачи
1. Запусти `/plan-feature описание фичи`
2. Дождись плана от architect — он сохранится сюда
3. Подтверди реализацию или скорректируй план

### Перед тем как сделать /clear
1. Запусти `/handoff`
2. Убедись что `plans/handoff.md` обновился
3. Теперь можно `/clear`

### Возобновление после /clear
1. Прочитай `plans/handoff.md` — там контекст прошлой сессии
2. Прочитай нужный план `plans/YYYY-MM-DD-*.md`
3. Продолжай с "Next immediate action" из handoff

## Что НЕ хранить здесь
- Финальный код (он в основных файлах проекта)
- Большие дампы данных
- Автогенерируемые файлы
